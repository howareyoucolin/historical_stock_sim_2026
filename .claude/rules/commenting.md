# Commenting Rules

- Add a brief purpose comment above each method or function definition so other
  developers can understand what it is for at a glance.
- If a logic block is complex, non-obvious, or constraint-heavy, add a short
  comment that explains the intent or reasoning behind it.
- Do not over-comment. Avoid line-by-line comments for obvious code or comments
  that only repeat what the code already says.
- Prefer clear naming and structure first, then use comments to explain intent,
  edge cases, or tricky behavior.
